1. REAL ON-CHAIN ARC ECONOMY (new: contracts/ + arc/)
   - contracts/src/ColonyForecastMarket.sol (+ test) — Foundry forecast/betting market, DEPLOYED to
     Arc testnet at 0xc40a8f2e29fe061cd4c0fe92cc73b9b43f9ada87. Ants stake real USDC, vote one
     outcome (group: home/draw/away; knockout: home/away), winners claim own stake + pro-rata share
     of losing pool minus treasury fee (default 1000 bps = 10%). Refunds if match canceled.
   - arc/ real payment rail (Tanguy):
       x402-agent-service.mjs / x402-agent-pay.mjs / x402-agent-common.mjs / x402-gateway-deposit.mjs
       / x402-transfer-status.mjs — REAL ant-to-ant x402 via Circle Gateway (EIP-3009 sign -> 402 ->
       pay -> 200). Seller mode = agent_wallet (payTo resolves from wallet store, not treasury escrow).
       Smoke tests passed with real Gateway transfer UUIDs (e.g. France/Senegal 2026-06-14,
       d1b46270-...; amount 0.0003 USDC; status received). Receipts under arc/receipts/.
       fund-agents.mjs — treasury seeds ant wallets (dry-run default, --broadcast to send).
       ledger-to-transfers.mjs — mirrors internal economy events (balance_update, internal_stake,
       payment_receipt, settlement_summary) into ONE native Arc USDC transfer per ant (nets deltas).
       forecast-market.mjs — deploy/create-market/stake CLI for the contract.
   - KNOWN SEAM (matches our own finding): Dynamic V3 MPC wallets can RECEIVE but cannot BUY via
     x402 yet — Circle GatewayClient needs an EOA private key; the 200 Dynamic wallets have no raw
     key, so buyer-side needs a Dynamic-backed BatchEvmSigner path, not GatewayClient({privateKey}).
   - Native vs ERC-20: arc scripts use native Arc USDC (18dp), not the 0x3600 ERC-20 (6dp) interface.

2. clickhouse_api/ — METERED KNOWLEDGE PLANE (new, FastAPI)
   Implements the three clickhouse rules: (1) timestamp gate (ts <= as_of_ts enforced in SQL AND
   re-checked per row in Python; test_gate.py proves no lookahead leak), (2) x402 metering (402->pay
   ->200), (3) premium tier for Worldcoin-verified humanId (discount + higher caps). Structured
   queries (no raw SQL/injection). This is the gated data pipeline we'd only scaffolded before.
